# Research Memory Ledger

This is the generic parent of project-specific research-memory skills.

## Memory Layers

Keep three layers distinct:
- execution lineage
- long-form interpretation
- durable claim memory

## When To Use

Use this skill when:
- a run just finished and should be recorded
- a note exists but the durable claim map is stale
- you need to review what is cleared, open, confounded, or falsified
- you want future branches to inherit prior learning

## Workflow

1. Read the current hypothesis surface.
2. Read the relevant note and run artifacts.
3. Decide whether the update is:
- new claim
- status change
- note-only
- rebuild without new claim
4. Append, do not rewrite.
5. Regenerate the current view.

## Guardrails

- Do not overwrite history to simplify the story.
- Do not turn a confounded result into a cleared finding.
- Do not confuse what ran with what was learned.

## Adaptation Notes

Specialize this skill by binding:
- the execution-manifest location
- the note location
- the claim-ledger format
- the current-hypothesis view
