**Supersedes:** experiments/mq_gora_v4, head_routing_backbone worktree

## 0. The Central Failure: Conditional Observation Without Conditional Contribution

The model has learned conditional observation, but not conditional contribution.

Every routing mechanism we have built — pi, beta, per-view kNN, cross-view attention,
disagreement regularization, geometry observers — improves how each head *sees* the data.
None of it changes how each head's *contribution to the final prediction* is weighted per row.

The router decides which head should inspect which view. The decoder still uses a row-invariant
mixing rule after flattening. So the architecture can discover row-specific structure and then
immediately erase the usefulness of that discovery at the prediction step.

```
encoder = dynamic, row-specific     ← routing shapes this correctly
decoder = static, row-agnostic      ← this is where the signal is lost
```

The model can say:
- "for this row, head 2 should trust GEO"
- "for that row, head 2 should trust SOCIO"

but the decoder still says:
- "I always use head 2's coordinates the same way"

That is the core signal-to-noise failure. Everything in this document is a fix to some part
of it, in order of expected leverage.

## 1. Current State — Honest Baseline

### What the backbone worktree got right

```
x_anchor[view]  (view-specific feature subsets — GEO=lat/lon, SOCIO=income, LOWRANK=PCA4)
  → HeadwiseEdgeBiasedViewEncoder[view]
     (per-head QKV, per-head geom_log_scale [H])
  → head_tokens [B, V, H, Dh]                     ← head structure preserved all the way through
  → HeadwiseCrossViewInteraction(iso, tokens, pi)  ← proper per-head cross-view attention
  → head_repr = (1-beta)*iso + beta*int  [B, H, Dh]
  → reshape [B, H*Dh]                              ← ★ head structure discarded here
  → RegressionHead MLP → pred [B]                  ← fixed combination, same for every row
```

HR_v1: RMSE=0.4833 | HR_v2_diverse: RMSE=0.4812 | G2_ref: RMSE=0.4546 | HGBR: RMSE=0.4433

### Why each piece underperforms

| layer | current code | what's wrong |
|-------|-------------|--------------|
| Observers | kappa/LID/LOF computed once in LOWRANK (PCA) space, applied to all views | GEO kappa and SOCIO kappa are different; router gets one blurred signal |
| Router input | `g [B, 7]` = [kappa, LID, LOF, density×4] | No per-view quality, no view-pair agreement, no label coherence |
| Beta prior | `beta = sigmoid(mode_head(g))` | No anchor to view agreement; trained only by indirect MSE gradient |
| **Prediction step** | `reshape [B, H*Dh]` → `Linear(H*Dh, 128)` → scalar | **Fixed weights: same mix for every row. All routing signal lost here.** |
| Temperature | `geom_log_scale [H]` fixed scalar per head | Same geometry bias for dense urban and sparse rural rows |
| Anti-collapse | cosine similarity on attention maps | Prevents representation collapse; ignores prediction collapse |
| Uncertainty | none | Cannot distinguish biased heads from uncertain heads |

## 2. Gap Map — 8 Original + 5 From Second Review

### Group A: Bad Routing Inputs

---

#### Gap 1 — Observer features are not view-specific *(Tier 1)*

**What's wrong:** kappa, LID, LOF are computed in LOWRANK (PCA4) space and used as a proxy
for all four views. GEO (2D: lat/lon) and SOCIO (5D: income/rooms/age) have completely
different manifold geometry. The router sees one blurred curvature and pretends it represents
all views — strong views cannot advertise their own local reliability.

**Fix:** compute kappa_v, LID_v, LOF_v independently in each view's own feature space.
Form a per-view quality score:
```
q_v = sigmoid(w1 * kappa_v_norm + w2 * (1/LOF_v_norm) + w3 * density_v_norm)
quality [B, V]
```
Inject as routing prior: `pi_logit_v += log(q_v + ε)`. Views with low curvature, low outlier
score, high density earn higher routing mass before any content attention.

**Normalization note:** kappa, LID, LOF, density live on very different scales.
All must be standardized using train-set statistics before entering the router.
Apply robust normalization (median/IQR) for LID and LOF which have heavy tails.

---

#### Gap 2 — Beta has no anchor to view agreement *(Tier 1)*

**What's wrong:** `beta = sigmoid(mode_head(g))` — beta is trained only by long, indirect
MSE backprop. Nothing in `g` tells the router whether views agree or conflict for this row.
agree_score was present in v2/v3 (≈0.11 for CA, ≈0.37 for MNIST) but was never wired to
the backbone router.

**Fix:** compute per-view-pair Jaccard overlap as a routing prior:
```
J(v1, v2) = |N_v1(i) ∩ N_v2(i)| / |N_v1(i) ∪ N_v2(i)|
J_flat [B, V*(V-1)/2]   — preserves coalition topology, not just mean
mean_jaccard [B]         — for beta prior
```
Beta prior: `beta_logit += w_beta * mean_jaccard`.
- High J → views say the same thing → interaction can synthesize safely
- Low J → views diverge → isolation protects the dominant signal

This gives beta a causal clue before gradient descent has to figure it out.

---

#### Gap 3 — Temperature is row-blind *(Tier 2)*

**What's wrong:** `geom_log_scale [H]` is a fixed scalar per head. A dense urban row
(many near GEO neighbors) needs different geometry bias strength than a sparse rural row.

**Fix:** make geometry strength row-conditioned:
```
geom_scale_h = softplus(w_h @ g_local + b_h)    [B, H]
```
One additional linear layer per head, conditioned on local density. This converts temperature
from a model constant into a local confidence controller.

---

### Group B: Good Routing, Weak Decision Logic

---

#### Gap 4 — Anti-collapse targets representations, not predictions *(Tier 2)*

**What's wrong:** `head_disagreement_loss` penalizes cosine similarity of attention maps.
Two heads can attend to completely different neighbors and still produce identical scalar
predictions. Representation diversity ≠ functional diversity.

**Three independent collapse channels, only one currently addressed:**
1. Representation collapse: cos-sim loss addresses this ✓
2. Routing collapse: all heads route to same view — not addressed
3. **Prediction collapse: all heads predict same scalar — not addressed**

**Fix:**
```
L_pdiv = -mean_B Var_h(pred_h)         # heads must differ in what they predict
L_lb   = -sum_h entropy(mean_B pi_h)   # load balancing: discourage one-view routing
```
`L_pdiv` is the sharper instrument — it aligns the regularizer directly with the business
outcome. Load balancing supports it.

---

#### Gap 5 — View consensus collapsed to scalar *(Tier 3)*

**What's wrong:** agree_score is one number per row. It loses the topology of agreement:
"all views agree", "two agree one disagrees", "one dominates" — these are different routing
situations requiring different fusion logic.

**Fix:** use `J_flat [B, V*(V-1)/2]` (from Gap 2) as router input. This gives the model
enough structure to detect view coalitions vs. outlier views vs. unanimous agreement.

---

### Group C: The Main Bottleneck — Decoder Credit Assignment Failure

---

#### Gap 6 — Regressor ignores routing variables *(Tier 1 — highest priority)*

**What's wrong:** the smoking-gun line:
```python
final_rep = head_repr.reshape(B, H * Dh)   # head identity becomes block of coordinates
pred = RegressionHead(final_rep)            # W[H*Dh, 128]: fixed mix for every row
```
`W[H*Dh → 128]` is equivalent to: `W[0:Dh,:] @ h0 + W[Dh:2Dh,:] @ h1 + ...`
This is a static combination. A row where head 2 correctly routed to GEO and a row where
head 2 correctly routed to SOCIO get the same downstream weights for head 2.

The MSE gradient cannot distinguish "head h was wrong because it routed badly" from
"head h was wrong because it was given the wrong downstream weight."

**The fix: Head-Gated Regressor**
```
head_repr [B, H, Dh]
  → pred_h  = HeadPred_h(head_repr[:, h, :])      [B, H]   H separate linear heads
  → gate_h  = softmax(GateNet(g, pi, beta, q))     [B, H]   routing-aware confidence
  → pred    = sum_h gate_h * pred_h                [B]      gated prediction
```

Why this is the highest-leverage change:
- MSE gradient flows directly to `gate_h`: if head h predicted poorly for rows in regime X,
  the gradient reduces gate_h for those rows and adjusts head_h's weights
- `gate_h` sees `pi` and `q`: the gate can learn "trust head h when its dominant view has
  high quality and the routing was peaked"
- This strictly generalises the current concat+MLP: when all gates are equal, it reduces to
  a weighted sum equivalent to a linear projection over head features
- Zero downside — this change can only help if the head representations are already diverse

**For classification:**
```
logit_h = HeadCls_h(head_repr_h)             [B, H, C]
logit   = sum_h gate_h.unsqueeze(-1) * logit_h  [B, C]
```

### Group D: Missing Structural Priors

---

#### Gap 7 — No table-structure prior *(Tier 3)*

**What's wrong:** view membership is implicit — features flow through `Linear(input_dim, rep_dim)`
with no identity signal. GEO is treated as "just another vector," not "spatial evidence."

**Fix:** learned view embedding token. Each view v gets `e_v [Dh]` added to the anchor
projection. The interaction branch can then learn "GEO and SOCIO are structurally complementary"
rather than having to infer it from feature correlations.

---

#### Gap 8 — No feature extraction diagnostic *(Tier 2)*

**What's wrong:** we are training end-to-end without ever testing whether the backbone
itself is the bottleneck or whether a better decoder would unlock existing representations.

**Fix:** after training the end-to-end model, freeze the backbone and fit HGBR on
`head_repr [B, H*Dh]`. Call this the "backbone probe."

- Probe RMSE ≈ end-to-end RMSE → backbone is the bottleneck; improve routing first
- Probe RMSE << end-to-end RMSE → decoder is wasting a good backbone; Gap 6 is the problem
- This diagnostic is the most important sanity check before adding routing complexity

---

### Group E: Five Additional Gaps From Second Review

---

#### Gap 9 — Heads have predictions but not uncertainty estimates *(Tier 2)*

**What's wrong:** a head can be wrong for two different reasons: it is biased, or it is
uncertain. The gated regressor treats both the same — it adjusts gate_h by the scalar
prediction error but gives no mechanism for a head to say "I do not know."

**Fix:** heteroscedastic head:
```
mu_h, logvar_h = UncertaintyHead_h(head_repr_h)
gate_h = softmax(GateNet(...))
pred   = sum_h gate_h * mu_h
```
Loss: `L_nll = sum_h gate_h * [(mu_h - y)^2 / exp(logvar_h) + logvar_h]`

High-uncertainty heads self-down-weight through logvar_h in addition to being penalized
by gate_h. This is especially valuable in sparse regions where some views are unstable.
HGBR cannot model predictive uncertainty per view — this is a genuine differentiator.

Implement as optional extension to HeadGatedRegressor. Add `use_uncertainty` flag.

---

#### Gap 10 — Routing and decoder gate can contradict *(Tier 2)*

**What's wrong:** after R4, it is possible for the router to say "head 2 is important" (high
pi mass on head 2's dominant view, high quality q for that view) while the decoder gate says
"head 2 is unimportant" (low gate_h for head 2). This inconsistency wastes capacity and
destabilizes training.

**Fix:** soft routing-gate consistency loss:
```
L_cons = mean_B KL( softmax(mean_v pi_h) || gate_h )
```
This does not force equality — a head can have high routing mass but low gate confidence,
which is legitimate if the head's representation is not predictively useful. But it penalizes
systematic divergence between what the router says is important and what the decoder trusts.

Weight lightly: `λ_cons ≈ 0.01`. Monitor as a diagnostic even if not used in loss.

---

#### Gap 11 — Head-gated ensemble has no global safety net *(Tier 2)*

**What's wrong:** if the head ensemble misfires — all heads specialise on local corrections
and miss a coarse global trend — the head-gated regressor has no fallback.

**Fix:** residual global path:
```
pred = (sum_h gate_h * pred_h) + w_global * GlobalResidual(mean_h(head_repr_h))
```
`GlobalResidual` is a small MLP on the mean-pooled head representation. `w_global` is a
learned scalar initialized to 0 so the global path starts as a zero residual and is only
used if the head ensemble underperforms.

This protects against catastrophic failure of the head-gated path and gives the model a
smooth fallback during early training before heads have specialized.

---

#### Gap 12 — Routing inputs are not normalized *(Tier 1 — operational)*

**What's wrong:** q, J_flat, sigma2_v, density, LID, LOF can all live on wildly different
numerical scales. If these enter the router raw, one variable can dominate routing logits
for scale reasons rather than signal content. Training becomes noisy.

**Required normalization discipline:**
```
kappa_v_norm    → standardize (mean/std over train set, per view)
LID_v_norm      → robust: log-transform then standardize (heavy-tail)
LOF_v_norm      → robust: clip at 99th percentile then standardize
density_v_norm  → already in [0,1] from current observers.py
J_flat_norm     → already in [0,1] by construction (Jaccard)
sigma2_v_norm   → robust: log-transform then standardize (target variance is heavy-tailed)
```

All normalizations are computed on the train set and applied to val/test using saved stats.
These are not optional — without this, router training will be dominated by scale artifacts.

---

#### Gap 13 — sigma2_v leakage discipline *(Tier 1 — operational)*

**What's wrong:** sigma2_v = Var(y_train[N_v(i)]) is in principle regression-safe (statistics,
not raw labels). But it requires strict boundary conditions to avoid leakage:

**Required leakage controls:**
1. Train rows: exclude self when computing N_v(i). Use leave-one-out kNN (already done in
   `views.py`'s `_query_reference_knn` with `query_idx` self-exclusion).
2. Val/test rows: N_v(i) is always from the train set only — guaranteed by train-referenced
   kNN construction.
3. sigma2_v enters only the router, never the encoder or regression head.
4. Log the range of sigma2_v across views as a sanity check. If any view has σ²≈0 everywhere,
   it is either trivially predictable or there is a leakage bug.

## 4. Revised Experiment Ladder — Step A/B/C/D

**Key insight from second review:** the original R0→R4 order tests improvements before
testing the decoder. The decisive test is *R4 first*. The probe *R8 second*. Then routing
improvements only if needed.

The real question is not just "does R4 beat G2?" but:
- Does R4 alone give a large jump over the pre-R4 baseline?
- Does R8 probe show the backbone was already strong before R4?
- After R4, do additional gains become smaller and more incremental?

If yes to all three: **the bottleneck was the decoder** — confirmed.
If R4 does not improve much: the backbone never learned differentiated head information,
or the router variables are too weak to support useful gating. Then diagnose before adding more.

---

### Step A — Isolate the decoder hypothesis

Run: `backbone_v5_ref` → `v5_head_gated_pred` (R4) on top of the current best backbone.
Do NOT change any routing inputs yet. Just replace reshape+MLP with HeadGatedRegressor.

Expected: RMSE drops significantly. If not, the head representations are not yet diverse
enough to gate usefully — move to Gap 1 first, then retry R4.

Gate: **R4 must beat G2=0.4546 to proceed.**

---

### Step B — Run backbone probe

Freeze the pre-R4 backbone. Fit HGBR on `head_repr [B, H*Dh]`. Compare probe RMSE to
end-to-end RMSE.

- Probe RMSE << end-to-end RMSE: confirms the backbone encoded signal that the decoder
  was discarding. Step A (R4) was the right fix.
- Probe RMSE ≈ end-to-end RMSE: backbone itself needs better routing information. Move
  to Gap 1+2 before R4 is meaningful.

This probe is **essential before claiming R4 credit**.

---

### Step C — If R4 helps but not enough, improve routing inputs

Add per-view quality (Gap 1) and Jaccard-anchored beta (Gap 2) with full normalization.
These give the router the information it needs to send better-differentiated representations
to the head-gated decoder.

Gate: RMSE < Step A result.

---

### Step D — Add loss regularizers and optional extensions

Prediction diversity (Gap 4), adaptive temperature (Gap 3), and optionally:
- Uncertainty head (Gap 9) — if heteroscedastic regression helps
- Global residual (Gap 11) — if head ensemble shows early instability
- Routing-gate consistency (Gap 10) — monitor first, add to loss only if divergence > 0.1

## 5. V5 Architecture — Complete Forward Pass

```
════════════════════════════════════════════════════════════════
PRECOMPUTE (offline, train-referenced)
════════════════════════════════════════════════════════════════

per_view_knn[v]    kNN in view v's feature space → idx[N,k], ew[N,k]
quality[B,V]       kappa_v, LID_v, LOF_v per view — normalized robustly
J[B,V,V]           Jaccard overlap between view neighborhoods
J_flat[B,V*(V-1)/2] upper-triangle of J
sigma2_v[B,V]      Var(y_train[N_v(i)]) — log-normalized, train-only
g[B,obs_dim]       global: kappa_global, LID, LOF, density[4]

════════════════════════════════════════════════════════════════
VIEW ENCODING (HeadwiseEdgeBiasedViewEncoderV5, per view)
════════════════════════════════════════════════════════════════

x_anchor_v + view_embed[v]        ← view membership token (Gap 7)
→ anchor_proj [B, H, Dh]

x_nei_v neighbors
→ nei_proj [B, k, H, Dh]

geom_scale_h = softplus(w_h @ g_local)  ← row-adaptive (Gap 3)
scores = QK/sqrt(Dh) + geom_scale_h * log(ew_v + ε)
attn_v [B, H, k]
head_tokens_v [B, H, Dh]

view_head_tokens = stack_v → [B, V, H, Dh]    ← structure preserved

════════════════════════════════════════════════════════════════
QUALITY-AWARE ROUTER (QualityAwareRouter)
════════════════════════════════════════════════════════════════

router_input = normalize([g, quality, J_flat, sigma2_v])   ← Gap 12 normalization
hidden = MLP(router_input)

pi[B,H,V]   = softmax(ViewHead_h(hidden) + log(quality + ε))   ← Gap 1 prior
beta[B,H]   = sigmoid(ModeHead_h(hidden) + w_b * mean_J)        ← Gap 2 prior
gate[B,H]   = softmax(GateHead(hidden, quality, pi, beta))      ← Gap 6 gate input

════════════════════════════════════════════════════════════════
ISOLATION + INTERACTION BRANCHES
════════════════════════════════════════════════════════════════

tokens_per_head = view_head_tokens.permute(0,2,1,3)  [B,H,V,Dh]

iso_h  = sum_v pi[h,v] * tokens[h,v]               [B,H,Dh]
int_h  = CrossViewAttn_h(iso_h, tokens[h], pi[h])  [B,H,Dh]
head_repr_h = (1-beta_h)*iso_h + beta_h*int_h       [B,H,Dh]

════════════════════════════════════════════════════════════════
HEAD-GATED REGRESSOR (HeadGatedRegressor) — replaces reshape+MLP
════════════════════════════════════════════════════════════════

    OPTION A (base):
    pred_h = Linear_h(head_repr_h)              [B,H]
    pred   = sum_h gate_h * pred_h + w_g * GlobalResidual(mean_h head_repr)  [B]   ← Gap 11

    OPTION B (+ uncertainty):
    mu_h, logvar_h = UncertaintyHead_h(head_repr_h)    [B,H], [B,H]   ← Gap 9
    pred   = sum_h gate_h * mu_h + w_g * GlobalResidual(...)  [B]

    FOR CLASSIFICATION:
    logit_h = ClassHead_h(head_repr_h)          [B,H,C]
    logit   = sum_h gate_h.unsqueeze(-1) * logit_h  [B,C]

════════════════════════════════════════════════════════════════
COMPOSITE LOSS
════════════════════════════════════════════════════════════════

    L_task   = Huber(pred, y)                          primary regression
    L_orth   = ||H^T H / ||H||_F - I||_F              head orthogonality
    L_lb     = -sum_h entropy(mean_B pi_h)             load balancing
    L_pdiv   = -mean_B Var_h(pred_h)                   prediction diversity   ← Gap 4
    L_nll    = sum_h gate_h * NLL(mu_h, logvar_h, y)   uncertainty (optional) ← Gap 9
    L_cons   = mean_B KL(softmax(mean_v pi_h) || gate_h)  routing-gate align  ← Gap 10

    L_total  = L_task
             + λ_orth * L_orth        (≈ 0.01)
             + λ_lb   * L_lb          (≈ 0.005)
             + λ_pdiv * L_pdiv        (≈ 0.01)
             + λ_cons * L_cons        (≈ 0.005, monitor first)
```

## 7. Key Interfaces

```python
# observers_v5.py
def compute_per_view_observers(view_feats, k, train_idx)
    → quality [N, V]              # kappa_v, LID_v, LOF_v in each view's space

def compute_jaccard(per_view_knn)
    → jaccard [N, V, V]           # Jaccard overlap; diagonal=1 guaranteed

def compute_label_variance_per_view(per_view_knn, y, train_idx)
```

## 9. Non-Negotiable Constraints

1. **V4 outputs not touched.** All v5 code lives under `experiments/head_routing_v5/`.

2. **sigma2_v is a routing prior only.** Never appears as an encoder input or regression
   feature. Strict leave-one-out for train rows. Saved train stats applied to val/test.

3. **Each ablation step is evaluated before the next begins.** A regressing step is
   documented as FAIL, not absorbed into the next baseline.

4. **Head collapse is measured by prediction correlation, not attention maps.**
   Gate C3: `Pearson(pred_h1, pred_h2) < 0.9` — two heads attending differently but
   predicting identically is still collapse.

5. **Gate C1 (A1 beating G2) blocks all further routing additions.** If the Head-Gated
   Regressor alone cannot beat G2, adding more routing inputs will not fix a decoder problem.
   Diagnose head representation diversity first (run backbone probe B1).

6. **All router inputs normalized on train set stats.** Saved normalization parameters
   applied to val/test. Never re-fit normalizers on val/test.

7. **Global residual initialized to near-zero contribution.** `w_global` initialized to
   a small constant (≈0.01) so the head-gated path does the work during early training.

## 11. The Two Hypotheses V5 Will Resolve

**Hypothesis A — Decoder was the bottleneck:**
The backbone already encodes sufficient signal. A1 (Head-Gated Regressor) alone gives a
large RMSE drop. B1 (backbone probe) confirms high-quality representations existed before A1.
Routing additions C1–C3 give smaller incremental gains.

**Hypothesis B — Encoder was underspecified:**
A1 alone gives a small improvement or none. B1 (backbone probe) shows the backbone never
learned differentiated head representations worth gating. Routing additions C1–C3 are required
before A1 can work.

The ablation ladder is designed to distinguish these unambiguously.
The correct diagnosis determines whether v6 should focus on decoder design (Hypothesis A)
or router and embedding redesign (Hypothesis B).
